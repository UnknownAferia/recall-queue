# Security Policy

Vora is currently in private alpha. Do not publish exploitable vulnerabilities,
Discord tokens, MongoDB credentials, private ticket transcripts or result
evidence in a public issue.

Report security problems through a private GitHub Security Advisory for this
repository or directly to the Vora server owner. Include the affected workflow,
impact and reproduction steps, but redact all player data and credentials.

If a secret is exposed, revoke it at the provider immediately. Deleting the
message or commit does not invalidate a leaked credential.
